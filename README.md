# Card-notification

## Описание проекта

Приложение, реализующее автоматическую рассылку уведомлений клиентам банка о скором истечении срока действия их банковских карт.

## Функциональность

### Управление клиентами

- Создание нового клиента с защитой от дубликатов
- Поиск клиентов по имени
- Удаление клиента

### Управление банковскими картами

Поддерживаются два варианта выпуска карты:

#### Ручной выпуск

- Ввод номера карты вручную
- Указание даты выпуска
- Указание срока действия
- Привязка к клиенту

#### Автоматический выпуск

- Выпуск карты по ID клиента
- Автоматическая генерация номера карты
- Автоматическое назначение:
    - даты выпуска — текущая дата
    - срока действия — 4 года

Дополнительно:

- Закрытие (аннулирование) карты
- Поиск карт по номеру
- Фильтрация карт по клиенту

### Уведомления по расписанию

- Отправка уведомлений по расписанию (либо таймеру)
- Поиск всех карт, срок действия которых истекает (например, через неделю)
- Генерация уведомления с выбором записи в файл, либо отправкой на почту

## Архитектура

Проект разделён на микросервисы, каждый из которых запускается в отдельном контейнере:

- **БД:** Postgres
- **Backend:** Java Spring
- **Frontend:** React

## Абстракция
С помощью application.properties (demo-module/src/resources) вы можете выбирать

- тип репозитория storage.type= 
  - sql (Postgres)
  - hashMap 
- Время отправки уведомлений cron.time = 0 0 0 * * *
- Тип присылаемых уведомлений cardNotifier.type=
  - file (запись в файл notification.txt)
  - mail (отправка на почту)
  - fileMail (запись в файл с одновременной отправкой на почту)

## REST API

## Клиенты

### GET `/rest/clients`
Получить список всех клиентов

### GET `/rest/clients?name={name}`
Поиск клиентов по имени (частичное совпадение, без учёта регистра)

### POST `/rest/clients`
Создать нового клиента

**Тело запроса:**
```json
{
  "fullName": "Иван Иванов",
  "birthDate": "1990-01-01",
  "email": "ivan@example.com"
}
```

> При дубликате (совпадении имени и даты рождения) возвращает существующего клиента

### GET `/rest/clients/{id}`
Получить клиента по ID

### DELETE `/rest/clients/{id}`
Удалить клиента по ID

---

## Карты

### GET `/rest/cards`
Получить список всех карт

### GET `/rest/cards?clientId={clientId}`
Получить все карты конкретного клиента

### GET `/rest/cards?number={number}`
Поиск карт по номеру (частичное совпадение)

### GET `/rest/cards/{id}`
Получить карту по ID

### POST `/rest/cards`
Добавить новую карту вручную

**Тело запроса:**
```json
{
  "cardNumber": "4200123412341234",
  "issueDate": "2024-03-30",
  "expDate": "2028-03-30",
  "clientId": 1
}
```

### POST `/rest/cards/issue/{clientId}`
Выпуск карты для клиента

> Генерирует номер карты, дата выпуска — текущая, срок действия — 4 года

### POST `/rest/cards/close/{id}`
Закрыть карту (аннулировать, не удаляя из БД)

### DELETE `/rest/cards/{id}`
Полностью удалить карту из БД


## Запуск

```bash
git clone https://github.com/VadimDun/Card-notification

cd Card-notification

# Необходимо добавить файл с локальными данными
cd demo-module/src/main/resources
```
создаем файл application-local.properties 
и добавляем в него следующий код, заменяя точки на свои данные 
```bash
spring.mail.username=..
spring.mail.password=..
```
После в корне проекта выполняем команду
```bash
docker-compose up --build
```

После запуска:

- **Frontend:** http://localhost:3000
- **Backend API:** http://localhost:8080/swagger-ui.html
